Guia de Git e GitHub

O que é Git e GitHub?

Git é um sistema de controle de versão distribuído que permite:

  • Rastrear mudanças no código fonte
  • Gerenciar diferentes versões do projeto
  • Trabalhar em equipe sem conflitos
  • Reverter para versões anteriores quando necessário
  • Criar branches para desenvolver features isoladamente

GitHub é uma plataforma de hospedagem de código que oferece:

  • Hospedagem de repositórios Git na nuvem
  • Interface web para gerenciar projetos
  • Ferramentas de colaboração como Pull Requests e Issues
  • Controle de acesso e permissões
  • Integração com outras ferramentas de desenvolvimento
  • Recursos para documentação como wikis e GitHub Pages

Juntos, Git e GitHub formam um ecossistema essencial para o desenvolvimento moderno de software, permitindo:

  • Colaboração eficiente entre equipes distribuídas
  • Versionamento seguro do código
  • Revisão de código através de Pull Requests
  • Rastreamento de problemas e melhorias

Fluxo de Trabalho

  1. Criar uma nova branch para sua feature: git checkout -b feature/nome-feature
  2. Verificar status das alterações: git status
  3. Adicionar todos os arquivos modificados ao commit: git add .
  4. Registrar as alterações com uma mensagem: git commit -m "mensagem"
  5. Enviar a branch para o repositório remoto: git push origin nome-da-branch
  6. Criar um Pull Request (PR) no GitHub para revisão e mesclagem das alterações.
  7. Resolver conflitos, se houver, e atualizar a branch conforme necessário.
  8. Mesclar o PR após aprovação.
  9. Atualizar a branch local com as últimas alterações do remoto: git pull origin nome-da-branch
  10. Deletar branch local após mesclagem: git branch -d nome-da-branch

Dicas:

  • Use mensagens de commit claras e descritivas
  • Mantenha commits pequenos e focados
  • Sempre teste suas alterações antes de fazer push
  • Atualize sua branch regularmente com a main/develop

Comandos Básicos

# Clonar repositório
git clone https://github.com/seu-usuario/seu-repo.git

# Entrar no repositório
cd seu-repo

# Criar e trocar para a branch develop
git checkout -b develop

# Criar uma nova branch de funcionalidade
git checkout -b feature/minha-feature

# Fazer alterações, adicionar e commitar
git add .
git commit -m "Minha primeira alteração"

# Enviar branch para o remoto
git push origin feature/minha-feature

# Verificar status do repositório
git status

# Ver histórico de commits
git log

# Desfazer alterações não commitadas
git checkout -- arquivo

# Atualizar repositório local
git pull origin main

# Listar todas as branches
git branch -a

# Merge de branches (estando na branch destino)
git merge nome-da-branch

Importante: Sempre verifique em qual branch está antes de começar a trabalhar usando git branch

Pull Requests (PRs)

No GitHub, após enviar sua branch, siga estes passos:

  1. Acesse o repositório no GitHub.
  2. Clique em Compare & pull request.
  3. Selecione a branch de destino (por exemplo develop).
  4. Descreva suas alterações e clique em Create pull request.

Você deve criar 3 PRs para develop (por exemplo, funcionalidades diferentes), depois uma PR de develop para main.

Resolvendo Conflitos

Conflitos acontecem quando dois commits alteram a mesma parte de um arquivo.

# Atualizar branch local
git pull origin develop

# Editar arquivos com conflitos
# (Git marcará com <<<<<<<, ======= e >>>>>>>)

# Após resolver manualmente os conflitos:
git add arquivo-com-conflito
git commit -m "Conflito resolvido"
git push origin develop

Dicas para Evitar Conflitos:

  • Mantenha sua branch atualizada com frequência
  • Comunique-se com a equipe sobre alterações em arquivos críticos
  • Divida grandes alterações em commits menores
  • Use rebase em vez de merge quando apropriado
